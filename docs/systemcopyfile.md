# systemcopyfile 使用说明（v1.0.1）

## 功能概述
- 选择源文件或整个文件夹，复制到目标文件夹
- 实时进度显示（进度条与百分比）
- 可随时取消复制，避免长时间阻塞
- 复制过程稳定，针对大文件采用 1MB 缓冲与进度节流
- 支持本地磁盘路径与网络共享路径（如 `\\server\share`）

## 运行环境
- Windows 10 及以上（已验证）
- .NET 8 运行时与 MAUI 工作负载
- 开发人员模式建议启用（用于本地调试运行）

## 安装与运行
1. 命令行构建：
   - `dotnet build -f net8.0-windows10.0.19041.0`
2. 运行可执行程序：
   - `c:\Users\DELL\Documents\trae_projects\007\systemcopyfile\bin\Debug\net8.0-windows10.0.19041.0\win10-x64\systemcopyfile.exe`
3. Visual Studio 运行：
   - 打开解决方案，选择目标为 `Windows Machine`，点击“播放”启动

## 使用说明
1. 点击“源文件/文件夹”后的“选择...”按钮，选择要复制的源文件夹
2. 点击“目标位置”后的“选择...”按钮，选择目标文件夹（可为本地或网络共享）
3. 点击“开始复制”启动任务；进度条与百分比实时更新
4. 复制过程中可点击“取消”快速终止任务
5. 状态栏将显示“复制进行中/完成/失败”等信息

## 路径与权限建议
- 网络共享路径示例：`\\192.168.1.10\shared\docs`
- 若目标为受保护目录，需使用有权限的账户运行或调整目标文件夹权限
- 建议在 SSD 上进行大量小文件复制以获得更好的体验

## 常见问题
- “未响应/卡顿”：已通过进度节流与大缓冲优化；如仍存在，可在网络盘上使用更高带宽或本地复制
- “复制失败”：检查源/目标路径存在性与访问权限；在状态栏查看提示
- “目标为网络共享慢”：建议在局域网高带宽环境下操作，并避免实时杀毒扫描同一目录

## 版本信息
